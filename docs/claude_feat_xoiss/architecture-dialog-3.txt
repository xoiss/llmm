  дописывала бы в конец файла новый блок без чтения и перезаписи всего документа. Это ограничение не библиотек, а природы формата: TOML — текстовый, без бинарного
  индекса, поэтому «дописать» структурно — значит сгенерировать текст и записать его в конец файла.

  Таким образом, вариантов три, и они те же, что и без поиска альтернатив:

  1. Ручной append — записать в конец файла текст вида \n[[messages]]\nrole = "..."\ncontent = "...", самостоятельно экранируя строки. По сути, это то, что в
  архитектуре сейчас. Экранирование несложное, но его нужно реализовать корректно.
  2. tomlkit load → modify → save — чисто, без ручного экранирования, O(n) на каждое сообщение. Для интерактивного чата это не проблема.
  3. Гибрид — tomlkit для создания файла (заголовок + начальные сообщения при rollback), ручной append для горячего пути (добавление одного сообщения в ходе
